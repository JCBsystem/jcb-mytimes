  <action>
Create `web/src/lib/auth.ts` with the following:

1. **AuthContext and AuthProvider:**
   - Create a React context `AuthContext` with shape:
     ```typescript
     interface AuthState {
       user: User | null;          // firebase/auth User
       projectKey: string | null;  // from customClaims.projectKey
       loading: boolean;           // true while onAuthStateChanged hasn't fired yet
     }
     ```
   - `AuthProvider` wraps children in `AuthContext.Provider`
   - On mount, subscribe to `onAuthStateChanged(auth, async (firebaseUser) => { ... })`
   - When firebaseUser is not null, call `getIdTokenResult(firebaseUser)` to read `claims.projectKey`
   - Set `user`, `projectKey` (string | null), and `loading: false`
   - When firebaseUser is null, set `user: null, projectKey: null, loading: false`
   - Return unsubscribe on cleanup

2. **useAuth hook:**
   - Returns `AuthState` plus helper functions:
     - `signUp(email: string, password: string): Promise<void>` -- calls `createUserWithEmailAndPassword(auth, email, password)`
     - `login(email: string, password: string): Promise<void>` -- calls `signInWithEmailAndPassword(auth, email, password)`
     - `logout(): Promise<void>` -- calls `signOut(auth)`
     - `createProject(): Promise<string>` -- calls `httpsCallable(functions, 'createProject')()`, then calls `auth.currentUser!.getIdToken(true)` to force token refresh so the new claim is picked up, then calls `getIdTokenResult(auth.currentUser!)` to read the new `projectKey` from claims, updates state, and returns the projectKey string

3. **Imports needed:**
   - `createUserWithEmailAndPassword`, `signInWithEmailAndPassword`, `signOut`, `onAuthStateChanged`, `getIdTokenResult` from `firebase/auth`
   - `httpsCallable` from `firebase/functions`
   - `auth`, `functions` from `@/lib/firebase`
   - `createContext`, `useContext`, `useEffect`, `useState`, `useCallback` from `react`
   - Type `User` from `firebase/auth`

4. **Critical implementation detail for createProject:**
   After the callable returns, the custom claim exists on the server but the client's ID token is stale. You MUST call `auth.currentUser!.getIdToken(true)` to force a token refresh BEFORE reading `getIdTokenResult`. Without this, `projectKey` will be null until the token naturally expires (~1 hour).
